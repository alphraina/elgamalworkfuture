⬇️ Download the Script (Admin Only)

Log in to the CMMS as Admin, then:

  1. In the left menu → scroll to the bottom → click Factory Settings
  2. Scroll to the bottom of Factory Settings
  3. Find the green section: "OMTP Log Pusher — Download Ready-to-Run Script"
  4. Click the big green Download Script button
  5. The file cmms_Start.bat will download — save it anywhere (e.g. Desktop)
✅ The downloaded file already contains your server URL and API key inside it — no editing needed.
📌 Where to find it in CMMS:
Left Menu → Factory Settings

2

🐍 Python — Installed Automatically (no action needed)

The script checks for Python when it runs. You do not need to install anything manually.

✅ Python already installed

The script detects it automatically and continues — even if it was not added to PATH during installation.

🔽 Python not found

The script automatically downloads Python 3.11 from python.org and installs it silently. No clicking, no checkboxes. Takes 1–2 minutes the first time only.

After installation the PATH is updated automatically in the current session — the script continues without needing a restart.

💡 Only needs internet connection the first time. Once Python is installed, it is permanent and this step is skipped on every future run.
3

▶️ Double-Click to Run — Zero Questions

Double-click cmms_Start.bat — a green window opens. It will:

  • Check Python — install it automatically if missing ✅
  • Add Python to PATH automatically ✅
  • Install required packages automatically ✅
  • Download the pusher script from CMMS automatically ✅
  • Auto-identify this PC via Machine Registry (IP address lookup) ✅
  • Auto-fetch the product model from today's shift setup in CMMS ✅
How identification works (two modes):

Mode 1 — IP Lookup (recommended)

When this PC's IP address is registered in CMMS → Machine Registry, the script calls the CMMS API to look up its own IP. CMMS returns the Line ID and today's product model from the shift setup automatically. No folder-name tricks needed.

Mode 2 — Folder-Name Fallback

If this PC's IP is not in the Machine Registry, the script reads the OMTP folder name instead:

TAM30202 → Line 2 TAM30203 → Line 3 TAM30210 → Line 10

💡 To use Mode 1: go to CMMS → Machine Registry, open the machine for this PC, and enter its IP address in the Machine IP field.

✅ When running, you will see messages like:
2026-03-22 08:15:00 [INFO] This PC: 192.168.1.105
2026-03-22 08:15:00 [INFO] Machine: SMT-01 (MR-042) — Line 2 — Model: TAM30202
2026-03-22 08:15:01 [INFO] Line 2 (Line 2) | TAM30202 — pushed 12 new rows
⚠️ Keep the green window open — do not close it. If you close it, data will stop sending.
4

📱 Team Leader: Enter Product Model Code Each Shift

The team leader must enter the product model code in CMMS so the script knows which CSV folder to read:

  1. Open CMMS → click Production Capacity
  2. Click the Shift Setup tab
  3. Find the correct line and type the Product Model Code (e.g. CPH2523)
  4. Click Save
📁 The script will then look for the CSV here:

D:\OMTP_LOG\CPH2523\data\22-3-2026.csv
⚠️ If the team leader hasn't entered the model code yet, the script will wait and show "No product model set." This is normal — just enter the code in CMMS and the script will continue automatically.

📅 Daily Usage (after first setup)

Zero configuration — the script identifies itself automatically and downloads the latest version every run.

  • Double-click cmms_Start.bat on the factory PC — starts immediately, no questions
  • The script auto-identifies this PC via IP lookup in Machine Registry

8

🚀 Auto-Start on Windows Startup (Background Mode)

On first run, the script asks: "Add to Windows Startup? (Y/N)" — type Y and press Enter.

What happens when you type Y:
  • Creates cmms_Startup.vbs in the script folder
  • Copies it to Windows Startup folder
  • Every time the PC turns on, it starts automatically
  • Runs completely in the background — no window
  • All output is saved to omtp_log.txt
What "background" means:
  • No green window on screen
  • PC works normally — no interruptions
  • Data is still pushed to Machine Monitor every 30 seconds
  • Open omtp_log.txt anytime to check status

📄 Reading the log file

Open omtp_log.txt (in the same folder as the .bat file) with Notepad to see what is happening:

2026-03-22 08:15:00 [INFO] OMTP Log Pusher starting — API: https://..., Line: 1 2026-03-22 08:15:01 [INFO] Reading: D:\OMTP_LOG\CPH2523\data\22-3-2026.csv 2026-03-22 08:15:01 [INFO] Pushed 12 new rows from CPH2523/22-3-2026 2026-03-22 08:15:31 [INFO] No new rows to push 2026-03-22 08:16:01 [INFO] Pushed 3 new rows from CPH2523/22-3-2026

🛑 How to stop the background process

  1. Press CTRL + ALT + DELETE on the keyboard
  2. Click Task Manager
  3. Click the Processes tab
  4. Find python.exe in the list
  5. Right-click it → End Task

🗑️ How to remove from Windows Startup

Delete the file cmms_Startup.vbs from this folder:

C:\Users\{YourUsername}\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\
💡 Quick way to open that folder: Press Win + R, type shell:startup and press Enter. Delete cmms_Startup.vbs from there.

🔄 If you download a new script (e.g. after API key change)

  1. Replace cmms_Start.bat in the same folder with the new downloaded file
  2. Run the new cmms_Start.bat once — it will ask about startup again
  3. Type Y to reinstall the background launcher with the new key
