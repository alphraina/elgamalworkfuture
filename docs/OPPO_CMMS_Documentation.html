ALL FEATURES

A comprehensive web + mobile application that centralizes all maintenance and production operations for a mobile phone manufacturing factory. It handles three production teams — Assembly, Test, and Packaging — across 8 production lines, with full role-based access control, real-time notifications, multilanguage support, and a mobile app wrapper.

👤 5 User Roles

Admin, Manager, Team Leader, Maintenance Tech, Inventory — each with specific permissions.

🏗️ 8 Production Lines

Assembly (8 lines), Test (8 lines), Packaging (8 lines) — tracked hourly.

🌐 3 Languages

English, Arabic (RTL), and Chinese — switchable at runtime without reload.

📱 PWA + APK

Works as a web app, installable PWA, or native Android APK via Expo.

🔴 Admin

Full access. Manages users, approves registrations, audit logs, system settings.

🔵 Manager

Cross-team operations. Creates tasks, approves vacation, views KPI and analysis.

🟢 Team Leader

Manages own team. Line plans, production, task assignment, vacation approval.

🟡 Maintenance

Downtime, broken machines, spare parts orders, PM tasks, training exams.

🟣 Inventory

Manages spare parts catalog and approves spare part orders. Work phone station. No exams or machine analysis access.

High-level operational overview tailored to the logged-in user's role.

  • Live counts for open downtime incidents, pending tasks, active broken machines
  • Spare part orders awaiting approval
  • Upcoming preventive maintenance (next 7 days)
  • Recent announcements from management
  • Quick navigation tiles to the most common actions for that role

Log every machine stoppage with full details for root cause analysis and reporting.

  • Fields: Machine name, machine code (scanned via QR), start time, end time, category (mechanical, electrical, software, other), root cause, resolution note, production line, shift
  • Duration is calculated automatically from start/end time
  • Public reports from unauthenticated users shown with amber "Public Report" badge
  • Filterable by date, shift, category, team, or machine
  • Audit logged on every create/update

Track machine breakdowns through their full lifecycle — from report to resolution.

  • Create: Machine name/code, description, priority (low/medium/high/critical), team, line
  • Assignment: Assign to a specific maintenance technician
  • Status tracking: Open → In Progress → Resolved → Closed
  • Resolution notes added when closing; color-coded priority badges
  • Filterable by status, team, and date

Schedule and track recurring maintenance tasks to prevent breakdowns. Displayed in a monthly calendar view.

  • Plan fields: Machine, task description, frequency (daily/weekly/monthly/quarterly), assigned technician, next due date
  • Monthly calendar view — tasks appear on their due date, color-coded by status
  • Click any day to see full task details in a panel below
  • Overdue tasks highlighted in red; completed in green
  • Filter by team and status

Complete spare parts catalog managed by the inventory team.

  • Fields: Part name, part number, category, quantity in stock, minimum stock level, unit, location, price
  • Low stock alerts: Parts below minimum level are flagged
  • Excel import/export: Bulk import from spreadsheet; export current catalog
  • QR codes: Each part displays a QR code for quick scanning
  • Full audit trail of all stock changes

Maintenance technicians request parts; inventory/admin team approves and fulfills.

  • Maintenance creates an order request (part, quantity, urgency, reason)
  • Inventory or admin approves/rejects with a note
  • On approval, quantity is deducted from stock automatically
  • Status flow: Pending → Approved / Rejected → Fulfilled
  • Notifications sent to the requester on status change

Work-order style task management with assignments and notifications.

  • Create: Title, description, priority (low/normal/high/critical), assignee, due date
  • Assigned user receives an in-app notification
  • Status: Pending → In Progress → Completed; completion note required
  • Team leaders only see tasks belonging to their team
  • Color-coded priority; overdue tasks highlighted

Daily shift planning for production lines — team leaders define who works on which line each day.

  • Select date, line, shift (day/night/morning/afternoon)
  • Assign operators to the line for that shift
  • View plans for the current week; switch between days
  • Team leaders can only plan for their own team's lines

Daily attendance tracking for all factory staff with automatic absent marking.

  • Record: Mark present/absent/late/half-day with check-in and check-out times
  • Shifts: Day, Night, Morning, Afternoon — each with configurable start time and max check-in window
  • Auto-absent: System automatically marks absent/leave 30 min after each shift's max check-in time — no manual action needed
  • Auto-leave: Users with approved vacation are automatically marked as Leave
  • Auto-cleanup: Attendance records older than 3 days are deleted automatically
  • Data feeds into KPI attendance score (35% weight)

Employee self-service leave management with an approval workflow.

  • Employee submits: start date, end date, reason
  • Manager or team leader reviews and approves/rejects with a note
  • Employee receives a notification on decision
  • Approved leave is reflected in attendance records

Central database of all factory machines with QR code tracking.

  • Fields: Machine name, code (unique), model, manufacturer, location, production line, team, status (active/inactive/under maintenance)
  • QR Code: Each machine has a printable QR code — scan from downtime or broken machine forms to auto-fill
  • Excel import/export: Bulk upload machines from spreadsheet
  • Filters by team, status, and line; full audit trail

Track which work phone and PCBA board is assigned to each employee.

  • Fields: Work ID, employee name, phone color, phone number / IMEI, PCBA number
  • PCBA number can be scanned using the device camera in-browser
  • Export Excel: Download current filtered view as .xlsx
  • Import Excel: Upload a spreadsheet to bulk-import with live preview and validation
  • Colors: Black, White, Blue, Green, Red, Gold, Silver, Purple, Pink, Orange, Brown

Advanced analytics engine for machine performance — upload test data CSVs and get instant insights.

  • Metrics: OEE, FPY (First Pass Yield), Pass / Retest / Escape rates, Average Cycle Time
  • Machine ranking: Risk score with CRITICAL / WARNING / GOOD classification
  • AI anomaly detection: Flags machines with unusual failure patterns
  • Charts: Failure frequency, Pareto, Escape rate trend, Cycle time distribution, OEE breakdown, Radar chart
  • CSV report download; paginated raw dataset viewer
Inventory role does not have access to Machine Analysis.

Schedule and manage training sessions for factory staff.

  • Create plan: Title, description, trainer, date/time, duration, team (or all), participants list
  • Team leaders only see plans for their team
  • Participants notified when added to a training plan
  • Status: Scheduled → In Progress → Completed / Cancelled

Create quizzes linked to training plans; record and grade employee results.

  • Create exam: Title, linked training, multiple-choice questions, passing score, time limit
  • Employees answer questions within the time limit
  • Grading: Automatic scoring; admin/manager can override grades
  • Pass/fail status with score %; detailed answer breakdown per question
  • Full exam history per employee
Inventory role does not have access to Training Exams.

Monthly performance scoring for every employee based on three weighted metrics.

MetricWeightCalculation

  • Color-coded: green (≥80%) / amber (60–79%) / red (<60%)
  • Click any employee row to open detail modal with metric breakdown
  • Filter by month, role, and team; export to CSV
  • KPI weight thresholds are configurable by admin

Broadcast messages from management to all or selected staff.

  • Create: title, message body, priority (info / warning / critical), active/inactive toggle
  • Priority levels use different color coding (blue / amber / red)
  • Inactive announcements hidden from staff but editable by admin/manager
  • Pinned at the top of the dashboard for all roles

Real-time in-app notification system plus optional push notifications for mobile users.

  • Bell icon in header shows unread count
  • Triggered by: task assignments, vacation decisions, spare part order status, PM due dates, training additions
  • Push notifications: Mobile app receives push notifications via Expo Push Notifications — users must grant permission
  • Mark individual or all notifications as read

Admin-only user administration panel with self-registration approval workflow.

  • Create user: Username, full name, password, role, team assignment, shift
  • Edit user: Change name, password, role, team, shift, extra roles
  • Pending approvals tab: Review self-registered users; assign role and team before approving or rejecting
  • Extra roles: A user can hold a secondary role in addition to their primary role
The admin account cannot be deleted or have its role changed — it is permanently locked.

Tamper-visible log of every significant system action — admin-only.

  • Tracks: create, update, delete actions across all modules
  • Each entry: timestamp, actor, action type, entity type, entity ID, entity label, change diff (before/after)
  • Filter by action type, entity type, date range, or actor
  • Expandable diff view shows exactly what changed
  • Admin can clear all logs (irreversible)

Site Settings

  • Toggle user self-registration (allow/block new users registering themselves)
  • Toggle public downtime reporting (allow unauthenticated machine issue reporting from login page)

Shift Settings

  • Configure start time and max check-in window for Day and Night shifts
  • Format: 24-hour (HH:MM) — e.g. Day shift starts 08:00, max check-in until 09:00
  • Auto-absent scheduler runs 30 min after each shift's max check-in time

When enabled by admin, any factory worker can report a machine issue without logging in.

  • Fields: name, team, machine name, machine QR code/ID (optional), description, production line
  • Reports appear in Downtime Records with a Public Report badge
  • Multilanguage: English / Arabic / Chinese

A React Native (Expo) wrapper that packages the full CMMS as a native Android app.

  • WebView-based wrapper pointing to the API server
  • Push notifications via Expo Push Notification service
  • Camera access for QR code scanning
  • File access for Excel upload/download
  • Build APK: eas build -p android --profile preview (requires EAS account)

The entire interface switches between three languages at runtime with no page reload.

  • English (EN) — default left-to-right layout
  • Arabic (AR) — full right-to-left (RTL) layout flip; all icons, spacing, and alignment mirror automatically
  • Chinese (ZH) — Simplified Chinese

Click the language toggle button in the header to cycle: EN → AR → ZH → EN. Preference stored in localStorage and persists across sessions.

Requires Node.js v20+, pnpm v9+, PostgreSQL 14+

1

Clone the repository

git clone <repository-url>
cd oppo-cmms
2

Create PostgreSQL database

psql -U postgres
CREATE DATABASE oppo_cmms;
\q
3

Create .env file

DATABASE_URL=postgresql://postgres:password@localhost:5432/oppo_cmms
SESSION_SECRET=change-this-to-a-long-random-string-32-chars-min
PORT=8080
NODE_ENV=development
4

Install dependencies

pnpm install
5

Push database schema

pnpm --filter @workspace/db run push
6

Seed sample data

pnpm --filter @workspace/scripts run seed
7

Start API server

pnpm --filter @workspace/api-server run dev

Runs on http://localhost:8080

8

Start frontend (new terminal)

pnpm --filter @workspace/cmms run dev

Open http://localhost:5173 in your browser.

  • Cannot connect to DB: Check that PostgreSQL is running and the DATABASE_URL in .env is correct. Run psql $DATABASE_URL to test the connection.
  • 401 Unauthorized on all API calls: The session cookie is missing. Log in first at /api/auth/login.
  • Frontend shows blank / white screen: Check browser console for errors; confirm the API server is running on port 8080.
  • Excel import fails: Ensure column headers match exactly: Work ID, Name, Phone Color, Phone Number, PCBA Number.
  • QR scan not working: Camera permission must be granted in the browser; HTTPS required in production.
  • Push notifications not arriving on mobile: Make sure the user granted notification permission in the Expo app and the API server can reach the Expo Push API.
  • Auto-absent not running: Confirm the API server started successfully — the scheduler logs "[Scheduler] Starting" on boot. Runs every 15 minutes.
